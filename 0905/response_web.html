response web이란?
하나의 html 문서로 pc, tablet, mobile 화면에서 최적화된 content를 보여주기 위한 기술이다.

response web에서는 가변크기 단위, 뷰포트 속성, 가로해상도별 서식을 적용하기 위한 미디어 쿼리속성이 사용된다.

response web site를 참고할 수 있는 사이트는 http://mediaqueri.es 가 대표적이다.

1. 가변크기 단위

  1. em - 부모요소의 font-size를 상속받아 크기가 결정된다.
  2. rem - root의 약자로서 부모의 font-size를 상속받지 않고 최상위(html)요소에 설정된 font-size를 기준으로 크기가 설정된다.
  3. vh - vertocal height 의 약자로서 web browser의 높이를 100으로 기준설정하고 크기가 결정된다.
  4. vw - vertical width 의 약자로서 web browser의 가로넓이를 100으로 기준설정하고 크기가 결정된다.
  5. vmin - 높이와 너비 중 짧은 쪽을 100으로 기준설정하여 크기가 결정된다.
  6. vmax - 높이와 너비 중 긴 쪽을 100으로 기준설정하여 크기가 결정된다.
  7. % - 부모요소를 기준으로 비율을 표현하는 단위이다. (response web에서 주로 사용.)

2. viewport 속성

사용자의 디바이스 화면크기에 맞추어 웹문서를 100%로 보여줄 수 있는 속성으로서 pc보다 tablet과 mobile user에게 필요하다.

[ 작성법 ]
head 영역에 meta name="viewport" content="width=device-width, initial-scale=1"내용을 삽입한다.
width=device-width :가로너비는 장치의 너비로 지정한다.
initial-scale=1 : 실제 화면 크기 비율을 맞춘다. 1=100%, 2=200% 확대이다.

3. mediaquery 속성

browser의 가로너비(해상도)에 따라 또는 사용자의 display 가로 해상도에 따라 css를 적용하기 위한 기술로서 반응형 webpage 제작시 필수 속성이다.

mediaquery 사용 시 각각의 디바이스 별 해상도에 대해 이해할 필요가 있는데, pc 홈페이지의 가로해상도는 최소 1024px 이상, tablet의 가로 해상도는 768px ~ 1023px, mobile의 가로해상도는 0px ~ 767px 까지로 영역을 잡는 것이 보통이다.

디바이스 해상도를 기준으로 정하는 지점을 'BreakPoint'라고 한다.

디바이스 별 해상도 기준 참고 : http://uhb.kr/book/wiki/rwd/resolution 

        [ 작성법 ]
        @media screen and (가로크기 해상도){
            적용할 css 서식
        }
